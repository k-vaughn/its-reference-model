General structure for reporting observed data. <br/><br/>Any PhysicalObject, as defined by the architecture, can report on a number of SensedValues. Each SensedValue is associated with a type (e.g., in this case ambientAirPressure) and a location (e.g., allows reporting of values at different locations when needed; in this example, "ambient" would always have location 0).<br/>The type of SensedValue dictates which specialization is used. In the case of ambientAirPressure, the specialized type of SensedValue used would be the SensedAmbientAirPressure. Based on the rules of specialization, this class automatically contains all of the attributes defined within its parents (e.g., value from SensedPressure and accuracy among others from SensedValue). <br/>The SensedValue is determined from 1 or more Sensors (which have various attributes not shown here).<br/><br/>While this example focuses on the SensedAmbientAirPressure, it should be able to serve as a general template for most readings of sensed phenomena.  I've even included vehicleHeading and vehicleSpeed - but I think Position needs to be handled separately due to differences in confidence and accuracy meta-data.<br/>
