Each specialization of a SensedValue should define a "value" attribute with the preferred units for the reading. In most cases, the preferred units should be based on SI-units with a decimal offset to allow an integer representation. For example, units might be metres, kilometres, centimetres, etc. and the preferred value would always be an integer. <br/>When converting values from an implementation using a physical data model that is based on different units, the value will need to be converted by a formula and the loss of accuracy will need to be reflected in the accuracy (e.g., if the PDM reports a value in meters (e.g., 5m) with an accuracy of +/- 1 meter, and the LDM prefers centimetres, the reported value would be multiplied by 100 (e.g., 500 cm) and the new accuracy would be adjusted to +/- 100 centimetres. <br/><br/>
